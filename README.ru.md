# Гаражный учет

[English version](README.md)

Небольшое self-hosted веб-приложение для учета домашних и гаражных расходников:
винтов, гаек, батареек, лент, клея, электрики и других вещей, которые обычно
исчезают ровно в тот момент, когда нужны.

## Возможности

- Веб-интерфейс с входом по паролю.
- Карточный вид инвентаря.
- Компактный списочный вид.
- Добавление, редактирование и удаление позиций.
- Учет количества, единицы измерения, категории, мест хранения, минимального остатка и заметок.
- Поиск по названию, категории, месту, QR/штрихкоду, проекту или заметке.
- Фильтр по категории.
- Фильтр по проекту или набору.
- Фильтр по месту хранения.
- Сортировка по названию, уровню запаса, количеству, месту или последнему изменению.
- Фильтр позиций, которые нужно пополнить.
- Быстрое пополнение и списание с настраиваемым шагом для каждой позиции.
- Отмена последних изменений остатка и удалений.
- История операций: пополнения, списания, создание и ручное изменение количества.
- Долгая история операций по каждой позиции в окне редактирования.
- Фото предмета или коробки, сохраненное вместе с позицией.
- Сжатие фото на стороне браузера перед сохранением.
- Поле QR/штрихкода для быстрого поиска.
- Сканирование QR и поддерживаемых штрихкодов камерой браузера, если доступно.
- Несколько мест хранения для одной позиции.
- Редакторы мест хранения и проектов для массового переименования или удаления.
- Печатный список покупок для позиций ниже минимального остатка.
- Светлая и темная темы.
- SQLite без отдельного сервера базы данных.

## Стек

- React
- TypeScript
- Vite
- Express
- встроенный в Node.js `node:sqlite`

Нужен Node.js 24 или новее, потому что backend использует `node:sqlite`.

## Локальный запуск

Установите зависимости:

```bash
npm install
```

Создайте файл окружения:

```bash
cp .env.example .env
```

Отредактируйте `.env` и задайте реальный пароль:

```env
PORT=8782
GARAGE_PASSWORD=change-me
```

Соберите frontend:

```bash
npm run build
```

Запустите приложение:

```bash
npm start
```

Откройте:

```text
http://localhost:8782
```

## Разработка

Запустите backend и Vite dev server вместе:

```bash
npm run dev
```

Vite dev server проксирует запросы `/api` на backend на порту `8782`.

## Хранение данных

SQLite база создается автоматически:

```text
data/garage.sqlite
```

Папка `data/` намеренно исключена из git.

## Деплой на домашний сервер

Простой вариант деплоя: держать приложение в пользовательской директории на
сервере:

```bash
cd ~/garage-inventory
npm ci
npm run build
npm start
```

Для постоянного фонового запуска можно использовать PM2:

```bash
pm2 start npm --name garage-inventory -- start
```

Приложение слушает порт из `.env`, например:

```text
http://192.168.1.82:8782
```

## Обновление развернутой копии

В директории приложения на сервере:

```bash
git pull
npm ci
npm run build
pm2 restart garage-inventory
```

## Переменные окружения

| Имя | Обязательно | По умолчанию | Описание |
| --- | --- | --- | --- |
| `PORT` | Нет | `8782` | HTTP-порт Express-сервера. |
| `GARAGE_PASSWORD` | Да | - | Пароль для входа в приложение. |

## Заметки

Проект рассчитан на доверенную домашнюю сеть или VPN. Если открывать его в
публичный интернет, лучше поставить HTTPS и подумать о более строгой
аутентификации.
